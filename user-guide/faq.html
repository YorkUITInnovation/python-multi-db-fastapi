Quick answers to common questions

Q: Do I need to know SQL to use this API?

A: No! For basic operations (insert, get, update, delete), you just need to specify the table name and data. The API generates the SQL automatically. SQL is only needed for the sqlExec endpoint for complex queries.

Q: Which databases can I connect to?

A: The API supports Oracle, MySQL, PostgreSQL, and MS SQL Server. Use the /connections endpoint to see which databases are configured on your server.

Q: How do I get an API key?

A: Contact your system administrator or IT department to obtain an API key. API keys are used to authenticate your requests and track usage.

Q: Can I update multiple records at once?

A: Yes! The updateRecord endpoint will update all records that match your WHERE conditions. The response tells you how many records were affected.

Q: Is there a limit on how many records I can retrieve?

A: Yes. The sqlExec endpoint has a maximum page size of 300 records. Use pagination to retrieve larger datasets page by page.

Q: Can I delete all records from a table?

A: No, for safety reasons. The deleteRecord endpoint requires a WHERE clause to prevent accidental mass deletions.

Q: What format should dates be in?

A: Send dates as strings in ISO format: "YYYY-MM-DD" (e.g., "2024-12-06"). For timestamps, use "YYYY-MM-DD HH:MM:SS".

Q: How do I know which server to use?

A: Use the /connections endpoint to see all available servers. If only one server is configured for a database type, you can leave the server parameter blank.
